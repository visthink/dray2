drae.ext.ghostscript

Routine for calling Ghostscript to produce background image. Depracted – will likely be removed in later version because DRAE will use Java-based rendering in the future.

generate-page-image

(generate-page-image pdf-file page-no)

Create a bitmap for a single page of a given PDF file using Ghostscript. Returns the exist code (0 = successful).

generate-page-images

(generate-page-images pdf-file)

Create a bitmap from a given page of the given PDF file using Ghostscript. Returns the exit code (0 = successful). First and last pages are optional entries for first and last page numbers.

ghostscript-path

(ghostscript-path)

Attempts to find and return the location of the Ghostscript executable. See also drae.util.exec/*additional-exec-paths* and drae.config/*gs-args*.

gs-page-output-filestring

(gs-page-output-filestring pdf page-no)

page-image-file

(page-image-file pdf n)

Returns (as a file) the image file for a particular page.

page-image-for

Runs Ghostscript on a single page and (when complete) returns the image file. Memoized.

page-image-for-direct

(page-image-for-direct pdf n)

Runs Ghostscript on a single page and (when complete) returns the image file. Not memoized.

use-ghostscript?

(use-ghostscript?)

True if Ghostscript is to be used (as indicated in the config file).
