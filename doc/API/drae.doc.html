drae.doc

Main routines for documents, pages, and visual elements. Contains visual element constructors and accessors, along with several routines for manipulating sets of visual elements.

bbox-union

(bbox-union & bboxes)

Create a new bounding box that contains all the given bounding boxes. May be a little slow.

common-text-style

(common-text-style text-style-list)

Given a sequence of text styles, return a text style with the values set for all common aspects, but with NIL for the non-common aspects.

full-bitmap-path-for

(full-bitmap-path-for vimage vdocument)

Given a VImage instance and its containing VDocument, return the full bitmap path for the image file representing the VImage instance.

get-vdocument

(get-vdocument pdf)

Return the VDocument record object for the given PDF file. Currently handles only text and text tokens.

make-bbox

(make-bbox x y width height)(make-bbox arg-list)

Creates instance of a bounding box object (a subclass of java.awt.geom.Rectangle2D.Double). Arguments can be numbers or strings that are parsable to numbers.

make-text-style

(make-text-style fname size bold italic color)(make-text-style fname size)

Create a text style object.

make-vdiagram

(make-vdiagram image items)

Create an instance of VDiagram from the given VImage instance and a set of items contained within the diagram.

make-vdocument

(make-vdocument filename items)

Create a single instance of a VDocument.

make-vimage

(make-vimage bitmap-path bbox)

Create a single instance of a VImage

make-vpage

(make-vpage page-no bbox items)

make-vtext

(make-vtext bbox tokens)

Make a single instance of a VText.

make-vtext-block

(make-vtext-block items)

Create a new text block.

make-vtext-token

(make-vtext-token text bbox style)

Make a single vtext token.

merge-adjacent-vtexts

multimethod

Given a set of vtexts (or a VPage or VDocument), look through all the visual elements and merge any adjcent pairs of vtexts.

merge-vtexts

(merge-vtexts vtext1 vtext2)

Merge the two vtexts, which are assumed to be in order. Merges all the tokens, and creates a style that is held in common.

p2x-prxml

(p2x-prxml pdf)

Returns the PRXML information for the given PDF by running it through pdftoxml, and reading in the result.

pages

(pages vdoc)

Collection of pages in a VDocument.

partition-vtext-by-whitespace

(partition-vtext-by-whitespace vtext margin)(partition-vtext-by-whitespace vtext)

Returns nil if no paritioning is needed.

prxml->vel

multimethod

Translate a single prxml node into a visual element object or nil (if not applicable).

prxml-vector-xml

(prxml-vector-xml href pdf)

Given a filename, return the XML from that vector file.

sort-horizontally

(sort-horizontally vel-list)

Given a list of visual elements with bounding boxes, sort the list spatially from left-to-right.

sort-vertically

(sort-vertically vel-list)

Given a list of visual elements with bounding boxes, sort the list spatially from top to bottom.

split-distant-vtexts

Review all the items in the given document and split those vtexts (i.e., text lines) that are likely to be close column cells. Uses the text height as a heuristic to find vtokens that are distant.

text-distance

(text-distance vel1 vel2)

Distance between right edge of first element and left edge of second.

texts-adjacent?

(texts-adjacent? txt1 txt2)

Heuristic test to see if two texts are adjacent. Rule is distance must be less than average character height.

valigned?

(valigned? vel1 vel2)

Are these two elements vertically aligned?
